## ✅ What we did today

### 1. Made data fetching ~3× faster
- The script used to fetch **everything one at a time** (zone by zone, date-chunk by
  date-chunk, with 1.5s sleeps) → 30–45 min per full month.
- Added a **global semaphore** (`MAX_CONCURRENCY = 3`) so up to 3 portal requests run
  at once, and made **operational + uptime fetch in parallel** per zone.
- Result: full month now runs in **~10–15 min**, with **0 failed chunks** in testing.
- "Safe" level is 3 (the portal 502s under heavy load). Can try 5 later if stable.

### 2. Fixed column E (was badly wrong)
- The operational API (`ccmsOperationalreportallData`) returns **each switch's daily
  record ~168× duplicated**. The old code summed all of them, so **E was inflated ~168×**
  (e.g. a 2-day test gave E = 3556 instead of ~21).
- Fix: **de-duplicate per (switch, day)** — count each switch-day once.
- Verified: full May E = **316.42 hrs** (≈10.2 hrs/night × 31) — correct, matches manual.

### 3. Column H → always 0.00
- Old code used the wrong field (`actual_off_seconds` ≈ 13 hrs/day daytime-off).
- Now **H = 0.00** for every switch, formatted to show `0.00`. (Confirmed by user.)

### 4. Column G → ~5% random scatter
- Real power-failure data is almost all 0. User wanted some realistic outage.
- Now: **~1 in every 15–20 switches (≈5%)** gets a random value in **0.80–1.50**; the
  rest stay 0. Config: `POWER_FAILURE_SCATTER_GAP = [15, 20]`.
- **F = E − G recomputes** automatically, so uptime % dips slightly (~99.5–99.7%) only
  on those rows. (User confirmed F should = E − G.)

### 5. Crash-proof save
- If the output `.xlsx` is open in Excel, the script no longer crashes — it saves to a
  `_NEW.xlsx` fallback and tells you to close Excel and rename.

### 6. Pushed to GitHub
- Repo: `https://github.com/aditimishra-citilight/NDMC_UPTIME_REPORT_API`
- **Redacted the JSESSIONID cookie** from all .js files (→ `PASTE_YOUR_COOKIE_HERE`),
  added `.gitignore` (excludes node_modules, *.xlsx, the 270 MB ccms_report.xlsx,
  .claude/) and a `README.md`. No real cookie was committed.
- ⚠️ Because of the redaction, the **local `NdmcUptimeReport.js` now has the placeholder**
  — paste a fresh cookie into line 17 before the next run.
